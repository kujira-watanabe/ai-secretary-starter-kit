# AI秘書スターターキット

自分専用のAI秘書を作るためのテンプレート一式です。Claude Code公式の標準フォルダ形式に準拠しています。

## 導入（受講生向け）

秘書の仕事場フォルダ（例：`~/ai-secretary`）を作り、その**空フォルダの中で** Claude Code を起動して、こう伝えます：

> このフォルダにスターターキットを展開して。次のコマンドを実行してOKです：
> `curl -sL https://github.com/kujira-watanabe/ai-secretary-starter-kit/archive/refs/heads/main.tar.gz | tar xz --strip-components=1`

Claude Code が確認を求めてきたら承認 → 展開完了です。`/exit` で一度出て `claude` で入り直し、「あなたは誰？」と聞いて自己紹介が返れば成功。

うまくいかない場合の代替（`git` で丸ごと取得）：

```bash
git clone https://github.com/kujira-watanabe/ai-secretary-starter-kit.git ai-secretary
cd ai-secretary && claude
```

## フォルダ構成

```
ai-secretary/
├─ CLAUDE.md            ← 秘書への基本指示（司令塔）
├─ .claude/             ← エンジン層：秘書の「働き方」の定義
│  ├─ settings.json     ← 権限設定（危険な操作のブロック）
│  ├─ rules/            ← 判断ルール（優先順位・承認・セキュリティ）※置くだけで自動で効く
│  ├─ skills/           ← 秘書に覚えさせた仕事の手順書
│  └─ agents/           ← サブエージェント（専門スタッフ）の定義
├─ context/             ← データ層：あなたの情報（変わりにくいもの）
├─ tasks/               ← タスクと今の状態（active.md／waiting.md など、毎日更新）
├─ inbox/               ← 秘書に渡す情報の置き場（メール文面・メモなど）
├─ outputs/             ← 秘書が作った成果物の置き場
├─ logs/                ← 実行・承認・改善の記録＋daily/（1日1ファイルの控え。上書きされず溜まる）
├─ templates/           ← 講座で使うシート類＋あなたのひな型置き場
└─ backup.sh            ← バックアップ（実行するだけ）
```

覚え方：**`CLAUDE.md`＋`.claude/`＝秘書の「働き方」（エンジン）、それ以外＝秘書の「記憶と仕事場」（データ）**。

- `.claude/rules/` の中のファイルは**置くだけで自動的に読まれます**
- `context/` や `tasks/` は自動では読まれないため、CLAUDE.md に「いつ何を参照するか」が書いてあります

## 入っているスキル（秘書ができる仕事）

| スキル | 何をする | 使う日 |
|--------|---------|--------|
| `/secretary-setup` | 事前課題③の引き継ぎシートを読み込んで初期設定 | Day1 |
| task-organize | タスク整理・優先順位・分解 | Day1〜 |
| inbox-intake | メール・メモ・依頼をタスクに変換 | Day2 |
| reply-draft | 返信案の作成（送信はしない） | Day2 |
| calendar-check | 予定と空き時間の確認 | Day2 |
| morning-briefing | 朝の計画づくり | Day3 |
| midday-replan | 日中の立て直し | Day3 |
| daily-review | 終業時の振り返り | Day3 |
| `_template/` | 新しいスキルを作るときの雛形 | Day2〜 |

## 入っているスタッフ（サブエージェント）

| スタッフ | 担当 | 使う日 |
|---------|------|--------|
| writer | 文章担当（返信文・案内文・依頼文） | Day3 |
| researcher | リサーチ担当（調査・比較・下調べ） | Day3 |
| reviewer | チェック担当（送信前・提出前の最終確認） | Day3 |

「writerに返信文を作らせて」「reviewerにチェックさせて」のように指名できます。

## templates/ に入っている講座シート

| シート | 使う日 |
|--------|--------|
| 引き継ぎシート_サンプル | Day1（事前課題③が無い人はこれで進める） |
| ClaudeCode基本操作チートシート | Day1〜毎日（手元に置く） |
| 判断シミュレーション_Day1 | Day1（引き継ぎ確認） |
| 宿題記録_Day2 | Day2→Day3の宿題 |
| AI秘書評価シート／ズレ分類早見表 | Day3（秘書の育て方） |
| 連携判断4軸シート | Day2（Gmail等の連携判断） |
| 自動化と権限モード／安全運用シート | Day3 |
| 自動実行手順_朝ブリーフィング | Day3（自動化1本だけ版） |

## はじめかた（Day1で一緒にやります）

1. **空の仕事部屋を作る**：エクスプローラー／Finderで、ホームに新規フォルダ `ai-secretary` を作成
   - Windows：エクスプローラーで「Linux → Ubuntu → home → ユーザー名」（見当たらなければアドレス欄に `\\wsl$`）
   - WSLが使えない方：`C:\Users\名前\ai-secretary` でもOK
2. **Cursorで開く**：左下「><」→WSLに接続 → `ai-secretary` を開く（まだ空でOK）→ ターミナルを開いて右側へ
3. **起動する**：右のターミナルで `claude`（信頼確認はフォルダ名を確認してからYes）
4. **キットを秘書に運び込ませる**：入力欄に貼り付け →
   「このURLからスターターキットをダウンロードして、このフォルダに展開してください: 配布URL」
   確認が出たら内容を読んでYes。左ペインにフォルダ構造が生えてきます
5. **入り直す**：`/exit` → `claude`（起動時にCLAUDE.mdを読むため）
6. **引き継ぐ**：事前課題③の引き継ぎシートを `inbox/` に置いて、次を実行

```
/secretary-setup
```

秘書があなたの情報を `context/` と `.claude/rules/` に整理して書き込みます。

## 大事な3原則

1. **必ずこのフォルダのルートで起動する**（サブフォルダの中で起動しない）
2. **機密情報（パスワード・APIキー・顧客の個人情報）はこのフォルダに置かない**
3. **1つの情報は1つのファイルに**（同じことを2か所に書かない。ズレの原因になります）

## 秘書の権限について

基本方針は「**読む、整理する、提案する。送信・削除・外部公開は人間が確認する**」。

- ルールでの禁止：`.claude/rules/approval-rules.md`・`security-rules.md`
- 技術的なブロック：`.claude/settings.json`（機密ファイルの読み取りと危険コマンドを deny 済み）
- Autoモード（確認なしで進めるモード）は、この settings.json がある状態で使うこと

## 育てる（フォルダを増やすのは必要になってから）

```
案件が3つを超えたら      → tasks/projects.md（案件一覧）
終わった仕事が邪魔なら    → archive/ を作る
関係者（人間）が増えたら  → context/people/（顧客・家族などを1人1ファイル）
AIスタッフを増やしたいなら → .claude/agents/（人はデータ層、スタッフはエンジン層）
定型文が溜まったら       → templates/ に自分のひな型を追加
案件が重くなったら       → 秘書の外に別プロジェクトフォルダを作る
```

最後の1つが大事：**秘書が持つのは案件の「状態」まで。案件の「中身」（成果物・顧客資料）は別プロジェクトに分ける。**

## バックアップ

週1回、ターミナルで：

```bash
./backup.sh
```

（バックアップ先は `backup.sh` の中の `DESTINATION` で変更できます）
