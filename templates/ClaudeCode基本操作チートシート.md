# Claude Code 基本操作チートシート

迷ったらこの2つだけ：**困ったら `/help`、「あれ？」と思ったら `Esc`**。

## 画面のセットアップ（Cursor × Claude Code）

1. Cursorでフォルダを開く（Windowsは左下の「><」マークからWSLに接続 → フォルダを開く）
2. メニュー ＞ ターミナル ＞ 新しいターミナル
3. CursorのAIパネルを × で閉じる（今日の相棒はClaude Code）
4. ターミナル上で右クリック →「位置の変更」で右側へ。枠の幅をドラッグで調整
5. レイアウト完成：**左＝フォルダ構造／中央＝エディタ／右＝Claude Code（会話）**

## 毎日使う操作

| やりたいこと | 操作 | ひとこと |
|-------------|------|---------|
| 起動する | ターミナルで `claude` | 秘書フォルダのルートが仕事場になっているか確認 |
| 仕事を頼む | 日本語で入力して Enter | 長い文章（メール本文など）は貼り付けでOK |
| 入力中に改行する | **Alt+Enter** | Enterだけだと送信されてしまう |
| コピー＆貼り付け | Ctrl+C ／ Ctrl+V | ターミナル内でも使える |
| ファイルを渡す | **ドラッグ＆ドロップ** | 入力欄にファイルのフルパスが入る |
| コマンドを使う | `/` で始めて入力 | `/init`・`/secretary-setup`・`/help`・`/clear`・`/exit` |
| 確認に答える | Yes／No を選ぶ | 内容を読んで選ぶ。**迷ったらNoにして質問する** |
| 止める | `Esc` | 作業を中断。何度押しても壊れない |
| 会話を仕切り直す | `/clear` | 新しい仕事の区切りで。ファイルは消えない |
| 終了する | `/exit` | 終了はこれに統一する |
| 続きから再開する | `claude --continue` | 前回の会話の続きから始まる |

> **できないこと**：この環境では**画像の貼り付けはできません**。スクショはファイルに保存してドラッグ＆ドロップで渡す。

## うっかり終了に注意（事故キー3つ）

| キー | 何が起きるか | 対処 |
|------|------------|------|
| `Ctrl+C` を素早く2回 | **突然終了する** | `claude --continue` で続きから復帰 |
| `Ctrl+D` を2回 | **突然終了する**（ターミナルごと閉じることも） | 同上。ターミナルが閉じたら開き直す |
| `Ctrl+Z` | 終了ではなく**一時停止**（固まったように見える） | ターミナルに `fg` と打つと戻る |

どれを押してしまっても会話は消えません。**`claude --continue` で続きから戻れる**ので慌てない。

## 少し慣れたら

| やりたいこと | 操作 | ひとこと |
|-------------|------|---------|
| ファイルを指名して読ませる | `@` を打ってファイル名を選ぶ | 例：「@tasks/active.md を見て優先順位を見直して」 |
| モードを切り替える | `Shift+Tab`（押すたびに切替） | 表示なし＝通常 → accept edits → plan → auto（詳細は `templates/自動化と権限モード.md`） |
| 過去に打った指示を呼び出す | `↑` キー | 同じ依頼を繰り返すときに |
| 環境の健康診断 | ターミナルで `claude doctor` | 起動がおかしいときの最初の一手 |

## 困ったときの順番

1. `Esc` で止める
2. 「今何をしようとしていた？」と聞く（状況を説明してくれる）
3. `/help` を見る
4. それでもダメなら `/exit` → `claude --continue` で入り直す
5. 起動自体がおかしいときはターミナルで `claude doctor`

> **安心材料**：Claude Codeがあなたの確認なしにメール送信・予定変更・ファイル削除をすることはありません（このキットのルールと権限設定でブロック済み）。
