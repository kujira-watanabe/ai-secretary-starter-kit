# 自動実行の設定手順（朝のブリーフィング1本だけ版・Day3）

秘書が「自分から動く」入口として、朝のブリーフィングを1本だけ自動化する。

## 前提

- `morning-briefing` Skill が動くこと（Day3午前までに確認済み）
- `.claude/settings.json` の deny が設定済みであること

## 方法A：半自動（まずはこちらを推奨）

毎朝、1コマンド打つだけにする。ターミナルの設定ファイル（`~/.bashrc` または `~/.zshrc`）に次の1行を追加：

```bash
alias asa='cd ~/ai-secretary && claude -p "/morning-briefing を実行して"'
```

追加したらターミナルを開き直し、朝は `asa` と打つだけ。
`claude -p` は「1回だけ実行して結果を表示する」モード（ヘッドレス実行）。

## 方法B：完全自動（cronで毎朝8時に実行）

```bash
crontab -e
```

を実行し、最後に次の1行を追加（平日8:00に実行、結果はログへ）：

```
0 8 * * 1-5 cd ~/ai-secretary && claude -p "/morning-briefing を実行して結果を outputs/ に保存して" >> logs/cron.log 2>&1
```

### WSLの注意

WSLはPC起動時にcronが動いていないことがある。動かない場合は：

```bash
sudo service cron start
```

それでも安定しない場合は方法Aで十分（「呼べば毎朝同じ品質で出る」だけで価値がある）。

## 守ること

- 自動化するのは ✅レベル（整理・提案・下書き）の仕事だけ（`templates/自動化と権限モード.md`）
- 実行結果は必ず `logs/` か `outputs/` に残す
- 1週間運用して問題がなければ、2本目（夜のレビュー）を検討する
