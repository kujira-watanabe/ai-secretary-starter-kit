# ズレ分類早見表（秘書の育て方）

秘書の回答がズレたら、「違う」で終わらせずに**分類→ファイルに反映**する。会話で直しても次回は忘れる。ファイルに書けば次回から効く。

| ズレの種類 | 例 | 反映先 |
|-----------|----|--------|
| 情報不足 | 案件の現在地を知らない | `tasks/`（active.md・projects系）や `context/` に追記 |
| 判断基準不足 | 優先順位が自分の感覚と違う | `.claude/rules/priority-rules.md` |
| 役割の誤解 | 秘書がやりすぎ／やらなすぎ | `CLAUDE.md`（役割・基本方針） |
| 出力形式の不一致 | 長すぎる・選択肢が多すぎる | `context/preferences.md` |
| 接し方のズレ | 正論だけ・命令口調 | `.claude/rules/operating-principles.md` |
| 権限の踏み越え | 確認なしで進めようとした | `.claude/rules/approval-rules.md`（＋必要なら `settings.json` の deny） |
| 今回だけの特殊事情 | 例外的な状況だった | ファイルは変えず、その場で指示するだけ |

## 反映のやり方

秘書自身にやらせるのが早い：

> さっきの回答はズレていた。原因は〇〇（上の分類）だと思う。
> どのファイルにどう追記すべきか提案して。承認したら書き込んで。
