# カレンダー確認

## 目的

予定と空き時間を把握し、タスクをいつ実行するかの候補を提案する。

## 入力

次のいずれか：

- Googleカレンダー連携（接続している場合）
- 利用者が貼り付けた今日・今週の予定
- `inbox/` に置かれた予定メモ

## 参照

- `tasks/active.md`（実行待ちタスクと所要時間）
- `context/constraints.md`（稼働時間・固定予定）

## 手順

1. 今日（または指定期間）の予定を確認する
2. 空き時間を洗い出す（移動・休憩も考慮する）
3. タスクの所要時間と照合する
4. 「このタスクはこの時間帯に」という実行候補を提案する

## 出力形式

- 今日の予定一覧
- 空き時間（合計と内訳）
- タスクの実行時間の提案（3件以内）

## 完了条件

利用者が「いつ何をやるか」を決められる状態。

## 承認が必要な操作

- 予定の作成（案の提示まで。登録は確認後）

## してはいけないこと

- 予定の変更・削除（提案もしない。気づいたことは報告のみ）
