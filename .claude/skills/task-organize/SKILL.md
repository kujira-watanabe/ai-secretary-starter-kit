# タスク整理

## 目的

散らかったタスクを整理し、「今日やること」と「最初の一手」まで具体化する。

## 入力

- `tasks/active.md` の現在のタスク
- 利用者から口頭・貼り付けで渡される未整理タスク

## 参照

- `.claude/rules/priority-rules.md`（優先順位の軸）
- `tasks/current-focus.md`（今週の重点・扱わないこと）
- `context/constraints.md`（稼働時間の制約）

## 手順

1. 未整理タスクを確認し、`tasks/active.md` の形式に整える
2. 各タスクの締切・重要度・所要時間を確認する（不明なら聞く）
3. `.claude/rules/priority-rules.md` に従って優先順位を付ける
4. 大きなタスク（2時間超）は、最初の15〜30分で始められる単位に分解する
5. 今日実行するタスクを3件以内に絞り、最重要事項を1つ決める
6. 保留・繰り越し・完了を整理する
7. `tasks/active.md` を更新する
8. 整理結果（今日の最重要事項・優先タスク）と `tasks/active.md` のスナップショットを `logs/daily/YYYY-MM-DD.md` に記録する（無ければ新規作成、あれば追記）。上書きされない控えとして残す

## 出力形式

- 今日の最重要事項（1つ）
- 優先タスク（3件以内、それぞれ最初の一手つき）
- 保留・繰り越しにしたもの（理由つき）

## 完了条件

利用者が「何から始めるか」を迷わず判断できる状態。

## 承認が必要な操作

- タスクの削除（提案のみ。削除は利用者の確認後）

## エラー時

締切や重要度が判断できないタスクは、勝手に順位を付けず「要確認」として分けて提示する。
