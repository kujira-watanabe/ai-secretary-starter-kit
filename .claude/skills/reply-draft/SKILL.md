# 返信案作成

## 目的

相手と目的に合った返信案を作り、利用者の判断材料とともに提示する。**送信はしない。**

## 入力

- 返信対象のメール・メッセージ（原文）
- 利用者の意向（あれば）

## 参照

- `context/preferences.md`（文体・トーンの好み）
- `context/roles.md`（どの立場で返信するか）

## 手順

1. 相手は誰か、返信の目的は何か（承諾・調整・断り・確認等）を確認する
2. 過去の文脈が `inbox/` にあれば確認する
3. 返信案を作成する
4. 判断が必要な点・不明な点を明示する
5. **送信せず、下書きとして提示する**

## 出力形式

- 返信案（そのままコピーできる形）
- 判断ポイント（あれば）：「◯◯は△△という前提で書いています。違えば教えてください」

## 完了条件

利用者が微修正だけで送信できる品質の下書きがある。

## 承認が必要な操作

- 送信（このスキルでは行わない。送信は常に利用者が行う）

## エラー時

相手との関係性や過去の経緯が不明で判断できない場合は、複数トーン（丁寧め／簡潔め）の2案を提示する。
