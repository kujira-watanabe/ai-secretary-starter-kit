# AI秘書 初期セットアップ

## 目的

事前課題③で作成した引き継ぎシートの内容を、AI秘書プロジェクトの適切なファイルに振り分けて書き込み、「自分の情報と判断基準を理解した秘書」の土台を完成させる。

## いつ使うか

- 初回セットアップ時（Day1）
- プロフィールや判断基準を大きく見直すとき

## 手順

1. 利用者に、事前課題③の引き継ぎシートのファイルの場所を聞く（`inbox/` に置いてもらうか、内容を貼り付けてもらう）。**まだシートが無い／未完成の場合は、`templates/引き継ぎシート_サンプル.md` を使って進める**（後で自分のシートができたら、再度このセットアップを実行して差し替える旨を伝える）
2. 引き継ぎシートを読み、次の対応表で振り分け先を決める：

| 引き継ぎシートの項目 | 書き込み先 |
|------------------|-----------|
| 1. 現在困っていること | `context/user-profile.md`（冒頭「秘書の存在理由」＋支援してほしい領域） |
| 2. 理想のAI秘書像 | `CLAUDE.md`（役割・基本方針への追記）＋`.claude/rules/operating-principles.md`（介入の型・声かけの仕方） |
| 3. 任せたい仕事 | `CLAUDE.md`（任せる仕事一覧）＋今後のスキル候補として提示 |
| 4. 自分の基本情報 | `context/user-profile.md`、`context/roles.md`、`context/goals.md` |
| 5. 自分の特性 | `context/preferences.md`（接し方）＋`.claude/rules/priority-rules.md`（タスク分解ルール） |
| 6. 優先順位の基準 | `.claude/rules/priority-rules.md`（「今回扱わないこと」は `.claude/rules/security-rules.md` へ） |
| 7. してほしくないこと | `.claude/rules/approval-rules.md`（🔴に追加）＋`.claude/rules/security-rules.md` |

> 振り分けの基準：**秘書の毎回の行動を縛るもの→`.claude/rules/`（置けば自動で効く）、判断の材料になる本人情報→`context/`（CLAUDE.mdの参照ルールで案内する）**

3. 書き込む前に、**振り分け案を一覧で提示して利用者の確認を取る**
4. 承認を得たら各ファイルに書き込む。各ファイルの「最終更新日」を今日の日付にする
5. 不足している情報（なべ式：目的・役割・前提・行動・出力・制約 の観点で欠けているもの）があれば、1〜2個ずつ質問して埋める
6. 現在のタスクを3〜5個聞き取り、`tasks/active.md` に登録する
7. 処理済みのシートを `inbox/done/` へ移動する（写した後はファイル側が正。二重管理しない）
8. **最後に、秘書の名前を決めてもらう**：「この秘書を何とお呼びしますか？」と聞き、決まった名前を `CLAUDE.md` の冒頭に「# 秘書の名前」として記録する。以後、その名前で自称・応答する
9. `templates/判断シミュレーション_Day1.md` を案内し、引き継ぎ確認を促す

## 守ること

- 特性（事実）と判断ルールを分けて書く。例：「集中が途切れやすい」は特性 →「大きなタスクは15〜30分単位に分解する」というルールに変換して `.claude/rules/` へ
- 顧客名などの固有名詞が含まれていたら、仮名（A社等）への置き換えを提案する
- 推測で埋めない。わからない項目は質問する

## 完了条件

- context/ と .claude/rules/ の全ファイルに利用者の情報が入っている
- `tasks/active.md` に現在のタスクが3件以上ある
- 処理済みのシートが `inbox/done/` に移動されている
- 秘書の名前が決まり、`CLAUDE.md` に記録されている
- 利用者に「今日やることを提案して」と試してもらい、妥当な提案が返る

## 完了後の一言

決めてもらった名前で自己紹介し、セットアップ完了を伝えて、次の一言を添える：
「試しに『今日やることを整理して』と話しかけてみてください」
