# 終業時レビュー

## 目的

今日を締めて、明日の自分に整理された状態を引き継ぐ。

## 入力

- 現在日時
- 利用者からの今日の報告（何が終わった・終わらなかった）

## 参照

- `tasks/current-focus.md`（今朝の計画）
- `tasks/active.md`、`tasks/waiting.md`

## 手順

1. 完了したことを確認し、労う（`context/preferences.md` のトーンで）
2. 未完了のことと、その理由を整理する（責めない）
3. 明日へ繰り越すもの／保留にするものを分ける
4. 返答待ちの状況を更新する
5. 明日の最重要事項の候補を1つ提案する
6. AI秘書自身の改善点（今日うまく支援できなかったこと）を1つ挙げる
7. `tasks/` の各ファイルを更新し、`logs/improvements.md` に改善点を記録する
8. 今日の振り返り結果を `logs/daily/YYYY-MM-DD.md` に追記する（朝のブリーフィングが作った同じ日付ファイル。無ければ新規作成）。上書きされない1日1ファイルの記録として残す

## 出力形式

```
■ 今日完了したこと
■ 未完了と繰り越し先
■ 保留にしたもの
■ 返答待ちの更新
■ 明日の最重要事項（候補）
■ 秘書の改善メモ
```

## 完了条件

明日の朝、morning-briefing がこの結果だけで計画を立てられる状態。

## 守ること

- 未完了を責める表現を使わない
- 改善点は会話で終わらせず、必要なら `.claude/rules/` や `.claude/skills/` への反映を提案する
